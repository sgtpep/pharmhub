Тестовое задание

Дано:
Есть несколько прайс-листов, доступных в виде файлов формата CSV. Примеры:
https://niko-opt.com/price_two.csv
http://opt.10x10.com.ua/opt10x10.csv
В каждом прайсе есть код товара (артикул), его название и цена.
Цели:
1. Написать Node.js сервер, который будет получать источник данных по его URL, парсить полученные данные из CSV, сохранять результаты в базе данных и возвращать в виде JSON по запросу.
2. Сервер должен отвечать на два типа запросов:
   1. GET /:shopName/:productId — возвращает JSON-объект следующего формата:
{
    "id": <ID товара>,
    "name": <название товара>,
    "price": <цена товара>
}
   1. POST /:shopName — запускает обновление прайса данного магазина.
1. Добавление новых магазинов должно осуществляться путём добавления файла конфигурации (JSON, YAML — на усмотрение разработчика). В конфигурационном файле должны указываться все необходимые опции: название магазина, ссылка на прайс, имена колонок для артикула, названия товара, его цены, используемые разделитель и тому подобное.
2. Данные должны записываться и храниться в БД, а не в виде файла на диске.
3. Написать простой UI, в котором можно совершаю следующие действия:
   1. Просматривать список магазинов.
   2. Просматривать страницу отдельного магазина, где выводится строка поиска, в которую можно ввести ID товара. После получения данных о цене товара эта цена выводится на этой же странице.
   3. Все переходы между страницами должны выполняться без перезагрузки страницы (т.е. приложение должно быть выполнено как SPA).
   4. Должна быть реализована минимальная авторизация путём отправки на сервер заголовка Authorization: API_KEY, не подписанные запросы должны режектиться.
1. В файле README.md описать, как настраивать и запускать продукт.
2. В проект должны быть включены unit-тесты (API-тесты).
Предлагаемый набор технологий:
Node.js >= 6.x
Express.js
MongoDB, Redis или PostgreSQL в качестве БД.
React/Angular/Vue для фронтенда на выбор.
